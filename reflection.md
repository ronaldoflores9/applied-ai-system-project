# PawPal+ — Reflection

## What this project taught me about AI and problem-solving

**Planning ahead is absolutely essential.** The biggest decision made during this project was the time spent analyzing and designing the system in advance using UML instead of immediately coding the scheduling logic. By the time implementation began, class boundaries had been defined, and the data flow was clearly outlined, which means that there wasn't any confusion regarding where the scheduling logic belongs. This helped considerably while trying to integrate the AI agent by allowing to connect the agent's commands directly to Python method calls rather than to spaghetti-like code.

**The best use of AI is when it helps to think.** The most helpful instances of interaction with AI during this project were those in which it helped to analyze trade-offs or generate options before making a certain decision. For example, when asking for the urgency score algorithm, the AI provided three candidates along with their complexity and integration considerations. Thus, instead of relying on a single solution, the decision had to be consciously made based on its pros and cons.

**You must validate AI output with your internal model of the program.** There was one time when implementing an AI-generated urgency equation led to priority values in which a new HIGH-priority task had higher precedence than an overdue LOW-priority task, contrary to my intentions. This problem only became apparent when testing a sample calculation for a specific set of numerical inputs. Otherwise, the scoring inversion might have remained undetected in the code.

**Safety precautions are inherent parts of the feature, not optional extras.** Validation of each input type (`validate_task_title`, `validate_time_hint`, etc.), as well as the limit of ten repetitions of the agentic loop, were initially viewed as cosmetic details. However, they proved useful by protecting the user interface from crashing, offering constructive feedback on errors, and giving me peace of mind about stability when dealing with invalid input. Implementing observability features such as logging along with functional features saved a lot of debugging time.

Conclusion: The most important takeaway from this essay is that **quality AI-driven design is essentially indistinguishable from quality engineering** — well-defined objectives, objective trade-offs, modular verification, and validated outcomes. AI expedites the process of converting concepts into executable programs but can never replace the human decision-making process that determines whether or not an idea should be developed.
